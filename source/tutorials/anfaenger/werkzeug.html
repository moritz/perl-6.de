Wenn man ein Programm schreiben will, muß der von Menschen lesbare Text, aus dem das Programm besteht ("Quelltext" oder "Source Code") irgend wie in etwas übersetzt werden, was der Computer versteht.

perl

Im Falle von Perl ist ein sogenannter "Interpreter" dafür zuständig, und den muß man sich installieren, um in Perl programmieren zu können.

Der Interpreter für Perl heißt "perl" (man beachte das kleine p am Anfang), und man findet ihn auf perl.org.

Windows

Windowsuser können sich von der Firma ActiveState ein perl-Paket mit Windows-Installer kostenlos herunuterladen. Die Installation sollte selbsterklärend sein.

Linux und Mac OS X

Benutzer von Linux und Mac OS X haben es sogar noch einfacher: auf ihren Systemen ist (fast immer) perl bereits installiert.

Andere Systeme

perl gibt es für fast jede Prozessorarchtiktur und jedes Betriebssystem, genaueres findet man auf Ports-Webseite von perl.

Texteditor

Die Programme der meisten Programmiersprachen, inklusive Perl, sind einfache Textdateien.

Damit sind keine formatierten Textdokumente gemeint, wie sie z.B. von Microsoft Word produziert werden, sondern einfache, undformatierte Textdateien.

Um diese zu bearbeiten, benötigt man einen Texteditor. Im Prinzip reicht dafür so etwas einfaches wie NotePad, doch es gibt auch komfortablere Varianten.

Sogenannte IDEs, "Integrated Development Environments" helfen, indem sie verschiedene Elemente des Programms farbig hervorrufen ("Syntax Hilighting"), das Programm auf Knopfdruck ausführen, Zeilen automatisch einrücken und bei Fehlern automatisch zu der entsprechenden Zeile im Programm springen.

Auf der Internetseite perl-communitiy.de gibt es eine gute Liste von geeigneten Editoren und IDEs, aus der man sich einen sympathischen aussuchen kann.
